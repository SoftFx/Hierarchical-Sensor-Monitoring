## Применяемые правила

### Основные принципы, которые я буду соблюдать:

1. **Думать перед кодированием**
   - Ясно формулировать предположения перед началом задачи
   - Задавать уточняющие вопросы при неясностях
   - Представлять альтернативные подходы, когда они существуют

2. **Простота прежде всего**
   - Реализовывать минимальный код, решающий задачу
   - Избегать избыточных абстракций и функций
   - Не добавлять "гибкость" или "конфигурируемость" без запроса

3. **Точечные изменения**
   - Изменять только код, непосредственно связанный с задачей
   - Не рефакторить несвязанный код
   - Соответствовать существующему стилю проекта

4. **Целеориентированное выполнение**
   - Определять четкие критерии успеха для каждой задачи
   - Проверять выполнение критериев перед завершением
   - Составлять план выполнения для многошаговых задач
