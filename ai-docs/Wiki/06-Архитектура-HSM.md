# Архитектура HSM

Основные модули:

* **HSM Server** — основной сервис проекта, является конечной точкой для пользовательских данных. Служит для обработки, валидации и хранения данных.
* **HSM objects** — API продукта, содержит описание, в каком виде значения должны поступать на HSM Server для корректной обработки.
* **DataCollector** — NuGet-пакет с классами для корректных запросов к серверу и различными реализациями датчиков для компьютера.
* **Wrapper** — обёртка DataCollector, написанная на C++, для использования в проектах на C++.

---

## Сервер

* **HSMServer** — веб-сервер для приёма данных от клиентов и управления клиентскими сайтами.
* **HSMServer.Core** — библиотека для хранения и пересчёта глобального состояния дерева датчиков.
* **HSMServer.Core.Monitoring** — библиотека для мониторинга сервера HSM.

---

## База данных

* **HSMDatabase** — менеджер базы данных, служит фасадом для запросов к БД.
* **HSMDatabase.AccessManager** — протокол формата сущностей БД, необходим для изоляции реализации базы данных.
* **HSMDatabase.LevelDB** — бинарный NoSQL-формат базы данных, изолирован от общей сборки и работает через AccessManager.

---

## LevelDB

* **EnvironmentData** — база данных с метаинформацией о структуре (пользователи, продукты, датчики, ключи, фильтры, настройки и т.д.)
* **SensorValues** — лёгкая база данных датчиков (введена после Спринта 8)
* **MonitoringData** — база данных старого формата (запись отключена после Спринта 8, чтение отключено после Спринта 12)
