# Конфигурация сервера

Сервер HSM настраивается через файл `Config/appsettings.json`, расположенный рядом с исполняемым файлом сервера. В Docker этот файл находится внутри контейнера по тому же относительному пути — смонтируйте директорию `Config/` для его сохранения.

Большинство настроек можно изменять во время работы через раздел **Configuration** в веб-интерфейсе без перезапуска сервера.

---

## Полный пример appsettings.json

```json
{
  "Kestrel": {
    "SensorPort": 44330,
    "SitePort": 44333
  },
  "ServerCertificate": {
    "CertificatePath": "",
    "CertificatePassword": ""
  },
  "BackupDatabase": {
    "IsEnabled": true,
    "PeriodHours": 1,
    "StoragePeriodDays": 10,
    "SftpConnectionConfig": {
      "IsEnabled": false,
      "Address": "",
      "Port": 22,
      "Username": "",
      "Password": "",
      "PrivateKeyFileName": "",
      "PrivateKey": "",
      "RootPath": ""
    }
  },
  "Telegram": {
    "BotName": "",
    "BotToken": "",
    "IsRunning": false
  }
}
```

---

## Порты

```json
"Kestrel": {
  "SensorPort": 44330,
  "SitePort": 44333
}
```

| Настройка | По умолчанию | Описание |
|---|---|---|
| `SensorPort` | 44330 | HTTPS-порт для приёма данных с датчиков (DataCollector и REST API) |
| `SitePort` | 44333 | HTTPS-порт для веб-интерфейса |

Чтобы изменить порты, обновите `appsettings.json` и перезапустите сервер, либо перейдите в `Configuration → Server` в веб-интерфейсе.

В Docker сопоставьте эти порты при запуске контейнера:

```bash
docker run -p 44330:44330 -p 44333:44333 ...
```

## TLS-сертификат

```json
"ServerCertificate": {
  "CertificatePath": "",
  "CertificatePassword": ""
}
```

По умолчанию HSM генерирует и использует самоподписанный сертификат. Чтобы использовать собственный `.pfx` сертификат:
1. Установите `CertificatePath` — абсолютный путь к файлу `.pfx`.
2. Установите `CertificatePassword` — пароль от сертификата.
3. Перезапустите сервер.

В Docker смонтируйте файл сертификата в контейнер:

```bash
docker run \
  -v /host/certs/hsm.pfx:/app/Config/hsm.pfx \
  ...
```

Затем установите `CertificatePath` в `/app/Config/hsm.pfx`.

## Резервное копирование базы данных

```json
"BackupDatabase": {
  "IsEnabled": true,
  "PeriodHours": 1,
  "StoragePeriodDays": 10,
  "SftpConnectionConfig": { ... }
}
```

| Настройка | По умолчанию | Описание |
|---|---|---|
| `IsEnabled` | `true` | Включить автоматическое периодическое резервное копирование |
| `PeriodHours` | `1` | Частота создания резервной копии (в часах) |
| `StoragePeriodDays` | `10` | Срок хранения локальных резервных копий перед удалением старых |

**Что копируется:** Базы данных `EnvironmentData` и `ServerLayout` — метаданные структуры, пользователи, продукты, датчики, панели мониторинга. История датчиков (база `History`) не включается в резервные копии из-за большого объёма.

Резервные копии сохраняются в директории `DatabasesBackups/`. В Docker смонтируйте эту директорию для хранения копий на хосте:

```bash
docker run -v /host/DatabasesBackups:/app/DatabasesBackups ...
```

### Мониторинг резервного копирования

Статус резервного копирования отображается в продукте HSM Server Monitoring под узлом `Backup/`:

| Датчик | Описание |
|---|---|
| `Backup/Local backup size` | Размер последней локальной резервной копии (МБ). Статус = `Error`, если копирование не удалось |
| `Backup/Remote backup size` | Размер последней загрузки на SFTP (МБ). Статус = `Error`, если загрузка не удалась |

Установите TTL-оповещение на эти датчики, чтобы получать уведомления, если создание резервных копий остановится.

### Загрузка по SFTP

Резервные копии могут автоматически загружаться на удалённый сервер через SFTP после создания каждой локальной копии.

```json
"SftpConnectionConfig": {
  "IsEnabled": true,
  "Address": "backup.example.com",
  "Port": 22,
  "Username": "hsm_backup",
  "Password": "your_password",
  "PrivateKeyFileName": "",
  "PrivateKey": "",
  "RootPath": "/backups/hsm"
}
```

| Настройка | Описание |
|---|---|
| `IsEnabled` | Включить загрузку по SFTP |
| `Address` | Имя хоста или IP-адрес SFTP-сервера |
| `Port` | Порт SFTP (по умолчанию: 22) |
| `Username` | Имя пользователя SFTP |
| `Password` | Аутентификация по паролю (оставьте пустым при использовании ключа) |
| `PrivateKeyFileName` | Путь к файлу приватного ключа (формат `.pem` или OpenSSH) |
| `PrivateKey` | Содержимое приватного ключа в виде строки (альтернатива `PrivateKeyFileName`) |
| `RootPath` | Удалённая директория для загрузки резервных копий |

Используйте либо `Password`, либо `PrivateKey`/`PrivateKeyFileName` для аутентификации — не оба одновременно.

## Telegram-бот

```json
"Telegram": {
  "BotName": "@YourBotName",
  "BotToken": "123456789:AAF...",
  "IsRunning": false
}
```

| Настройка | Описание |
|---|---|
| `BotName` | Имя пользователя Telegram-бота (с `@` или без) |
| `BotToken` | Токен, полученный от `@BotFather` |
| `IsRunning` | Если `true`, бот запускается автоматически при старте сервера |

Вы также можете запускать/останавливать бота во время работы через `Configuration → Telegram` в веб-интерфейсе без редактирования файла.

Полное руководство по настройке Telegram: [Настройка Telegram](14-Настройка-Telegram.md).

## Монтирование томов Docker

Всегда монтируйте эти директории для сохранения данных при перезапусках и обновлениях контейнера:

```bash
docker run \
  -v /host/Logs:/app/Logs \
  -v /host/Config:/app/Config \
  -v /host/Databases:/app/Databases \
  -v /host/DatabasesBackups:/app/DatabasesBackups \
  -p 44330:44330 \
  -p 44333:44333 \
  softfx/hsm-server:latest
```

Или в `docker-compose.yml`:

```yaml
services:
  hsm:
    image: softfx/hsm-server:latest
    ports:
      - "44330:44330"
      - "44333:44333"
    volumes:
      - ./Logs:/app/Logs
      - ./Config:/app/Config
      - ./Databases:/app/Databases
      - ./DatabasesBackups:/app/DatabasesBackups
```

| Директория | Содержимое |
|---|---|
| `Logs/` | Логи приложения |
| `Config/` | `appsettings.json` и TLS-сертификат |
| `Databases/` | Базы данных LevelDB (история датчиков, метаданные, панели мониторинга) |
| `DatabasesBackups/` | Автоматические резервные копии баз данных |

> ⚠️ Если вы не смонтируете `Databases/`, все данные датчиков будут утеряны при удалении или обновлении контейнера.
