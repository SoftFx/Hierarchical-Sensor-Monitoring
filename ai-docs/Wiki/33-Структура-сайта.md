# Структура сайта HSM

Сайт HSM — основной продукт для визуализации данных, поступающих на сервер. Он имеет следующую структуру:

* **Tree** (Дерево);
  * **Work with tree** (Работа с деревем);
  * **Data filtering** (Фильтрация данных);
  * **Graph and data table** (График и таблица данных);
* **Products tab** (Вкладка «Продукты»);
  * **Folders** (Папки);
  * **Products** (Продукты);
  * **Examples of connecting external services** (Примеры подключения внешних сервисов);
* **Access keys tab** (Вкладка «Ключи доступа»);
* **Users tab** (Вкладка «Пользователи»);
* **Configuration tab** (Вкладка «Конфигурация»);
* **API**.

---

## Tree (Дерево)

### Work with tree

Древовидная структура используется для отображения информации о продуктах, подключённых клиентами к HSM. Она визуально напоминает перевёрнутое дерево и отображает все продукты, узлы и датчики.

Каждый элемент имеет 4 возможных статуса (порядок отражает приоритет от низкого к высокому):

* **OffTime** — данные недоступны из-за настроек расписания на клиенте или сервере;
* **Ok** — полученные данные соответствуют ожидаемым;
* **Error** — полученные данные содержат ошибку.

Через контекстное меню (правый клик) пользователь может настраивать оповещения, игнорировать датчик или редактировать метаданные продукта. Доступные параметры зависят от назначенных прав.

### Data filtering

Для удобства работы используются настраиваемые фильтры (вызываются кнопкой `Filters`). Окно настроек появляется слева от дерева.

Доступные параметры:

1. **Update tree interval** — интервал обновления статусов в дереве. По умолчанию = 5 секунд.
2. **Status** — отображение датчиков с выбранным статусом. Если выбранный статус имеет более низкий приоритет, чем фактический на дереве, в продуктах и узлах будет указан статус с более высоким приоритетом (например, `Error`), чтобы избежать пропуска ошибок при мониторинге.
3. **History** — добавление в список продуктов без данных (не поддерживается).
4. **Notification** — отображение датчиков с включёнными уведомлениями (не поддерживается).
5. **Blocked** — отображение заблокированных продуктов (не поддерживается, функционал удаляется).
6. **Sort By** — порядок сортировки:
   * **By Name** — по алфавиту (A–Z). Используется по умолчанию;
   * **By Last Update** — по времени последнего обновления (чем ближе к текущему моменту, тем выше в списке).
7. **Apply** — подтверждение и применение выбранных фильтров.

После применения появляются подсказки с информацией о текущих фильтрах и их количестве.

### Graph and data table

Справа от дерева данные отображаются в виде графика или таблицы. Вид зависит от наличия подпродуктов/датчиков.

* **Если выбран узел с подпродуктами**, справа отображаются:
  * Имя продукта;
  * Текущий параметр `Expected Update interval` (можно редактировать);
  * Узлы/подпродукты и их состояния в виде `Grid` или `List`.
* **Если выбран датчик** (узел без подпродуктов), справа отображаются:
  1. Метаинформация датчика (просматривается по клику на `Show meta info`):
     * `Product` (родительский продукт)
     * `Path` (путь)
     * тип датчика
     * `Expected update Time` (по умолчанию от родителя)
     * `Description`
     * `Unit` (единица измерения)
  2. Краткая информация: статус, последние данные, время последнего обновления;
  3. Период времени для отображения данных;
  4. Количество баров (только для датчиков, собирающих информацию барами);
  5. Данные в виде `Graph` или `Table` (по умолчанию отображается 100 записей).

## Products tab (Вкладка «Продукты»)

Вкладка предназначена для создания, редактирования, удаления и просмотра папок/продуктов.

### Folders (Папки)

При наличии прав папка добавляется по ссылке `+Add folder`. Указываются имя и описание, после чего доступен переход на страницу редактирования. Удаление выполняется по ссылке `Remove`. При удалении папки её продукты перемещаются в раздел `Other products`.

**Настройки папки:**

* **General** — имя, описание, цвет, автор, дата создания, список подключённых продуктов. Добавление через `+Add product(s)`, исключение через `Exclude from folder`.
* **Settings** — `Keep sensors history` (срок хранения истории датчиков), `Remove sensors after inactivity` (удаление датчиков после периода неактивности). По умолчанию 1 месяц. Параметр `Time to sensors live` используется для `TTL alert` (по умолчанию `Never`).
* **Telegram** — информация о подключённых Telegram-чатах. Доступно добавление существующего или нового чата. Действия: `View/Edit`, `Remove from folder`, `Send test message`, `Go to chat`. На странице редактирования чата доступен тоггл `Enable messages` и параметр `Messages delay` (период агрегации уведомлений).
* **Users** — список пользователей и их ролей. Добавление через `+Add user`, управление ролями/удаление через `Edit`/`Remove`.

### Products (Продукты)

Добавление выполняется по ссылке `+Add product`. Список содержит:

1. **Name** — имя продукта;
2. **Managers** — менеджер продукта;
3. **Last Update** — время последнего обновления данных;
4. **Actions** — редактирование или удаление.

Доступен поиск/фильтрация одновременно по параметрам `Name` и `Managers`.

### Product editing (Редактирование продукта)

Форма открывается кликом по имени продукта или через `Edit` в контекстном меню. Позволяет редактировать `Name`, `Description`, управлять участниками и ключами доступа. Для добавления пользователя `Admin`/`Product Manager` выбирает логин, роль (`ProductManager` или `ProductViewer`) и нажимает `Add`. После этого пользователь появляется в списке `Members`, где можно изменить его роль или отозвать доступ.

### Examples of connecting external services (Примеры подключения внешних сервисов)

Подключение внешних продуктов к HSM выполняется с использованием ключа доступа продукта и адреса сервера.

* Подключение TTS выполняется через TTA;
* Подключение aggregator и adminEye выполняется через adminEye.

## Access keys (Ключи доступа)

Список ключей, позволяющих отправлять/читать данные, создавать узлы/датчики. Содержимое списка:

* **Name** — имя ключа (при создании продукта автоматически создаётся `DefaultKey`);
* **Key** — значение ключа;
* **Copy** — копирование в буфер обмена;
* **Permissions** — права:
  * `CanSendSensorData` — отправка данных датчиков;
  * `CanReadSensorData` — чтение данных датчиков;
  * `CanAddNodes` — создание узлов;
  * `CanAddSensors` — создание датчиков;
  * `Full` — включает все перечисленные права.
* **State** — статус ключа: `Active` (доступен), `Expired` (истёк срок), `Blocked` (заблокирован вручную).
* **Actions** — `Edit` (редактирование), `Block/Unblock` (ручная блокировка/разблокировка), `Delete` (удаление).

**Для добавления ключа** нажмите `Add key`. Форма добавления содержит:

* **Display name** — отображаемое имя ключа;
* **Permissions** — набор прав;
* **Expiration** — срок действия (задаётся при создании, недоступен для редактирования).
