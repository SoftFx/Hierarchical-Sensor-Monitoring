# Установка

Сервер HSM распространяется как Docker-образ. На этой странице описаны все способы развёртывания.

---

## Предварительные требования

* Docker установлен и запущен
* Порты `44330` и `44333` доступны на хосте

---

## Способ 1 — Docker Compose (рекомендуется)

1. Скачайте файл compose:
   ```bash
   curl -O https://raw.githubusercontent.com/SoftFx/Hierarchical-Sensor-Monitoring/master/docker-compose.yml
   ```
   Или создайте вручную:
   ```yaml
   services:
     app:
       image: 'hsmonitoring/hierarchical_sensor_monitoring:latest'
       restart: unless-stopped
       user: '0'
       ports:
         - '44330:44330'
         - '44333:44333'
       volumes:
         - ./Logs:/app/Logs
         - ./Config:/app/Config
         - ./Databases:/app/Databases
         - ./DatabasesBackups:/app/DatabasesBackups
   ```

2. Запустите сервер:
   ```bash
   docker compose up -d
   ```

3. Откройте веб-интерфейс:
   Перейдите на `https://localhost:44333`. Примите предупреждение о самоподписанном сертификате.
   
   Учётные данные по умолчанию: логин `default`, пароль `default`. Немедленно смените пароль.

---

## Способ 2 — Docker Run (вручную)

1. Загрузите образ:
   ```bash
   docker pull hsmonitoring/hierarchical_sensor_monitoring:latest
   ```

2. Запустите контейнер:
   ```bash
   docker run -u 0 -d \
     --restart unless-stopped \
     -v /host/path/Logs:/app/Logs \
     -v /host/path/Config:/app/Config \
     -v /host/path/Databases:/app/Databases \
     -v /host/path/DatabasesBackups:/app/DatabasesBackups \
     -p 44330:44330 \
     -p 44333:44333 \
     hsmonitoring/hierarchical_sensor_monitoring:latest
   ```
   Замените `/host/path/` на актуальный каталог на вашей машине.

   **Пример для Windows:**
   ```cmd
   docker run -u 0 -d ^
     --restart unless-stopped ^
     -v C:\HSM\Logs:/app/Logs ^
     -v C:\HSM\Config:/app/Config ^
     -v C:\HSM\Databases:/app/Databases ^
     -v C:\HSM\DatabasesBackups:/app/DatabasesBackups ^
     -p 44330:44330 ^
     -p 44333:44333 ^
     hsmonitoring/hierarchical_sensor_monitoring:latest
   ```

## Способ 3 — Скрипт

Готовые скрипты доступны в репозитории:

**PowerShell (Windows):**
Скачайте и запустите `docker_scripts/HSMserver/server_load.ps1`
```powershell
.\server_load.ps1
```

**Bash (Linux/macOS):**
Скачайте и запустите `docker_scripts/HSMserver/load.sh`
```bash
chmod +x load.sh && ./load.sh
```
Скрипты загружают актуальный образ и запускают контейнер с правильными маппингами томов и портов.

---

## Маппинг томов — важно

Всегда монтируйте эти четыре каталога. Без них все данные будут потеряны при остановке или обновлении контейнера:

| Путь на хосте            | Путь в контейнере           | Содержимое                          |
|----------------------|--------------------------|-----------------------------------|
| `./Logs`             | `/app/Logs`              | Логи приложения                  |
| `./Config`           | `/app/Config`            | `appsettings.json`, TLS-сертификаты |
| `./Databases`        | `/app/Databases`         | Все данные датчиков (LevelDB)         |
| `./DatabasesBackups` | `/app/DatabasesBackups`  | Автоматические резервные копии БД        |

> **Примечание:** Том `Databases` — самый критичный. Он содержит всю историю датчиков и конфигурацию. Никогда не удаляйте его без резервной копии.

## Порты

| Порт    | Протокол | Назначение                                                    |
|---------|----------|------------------------------------------------------------|
| `44330` | HTTPS    | Приём данных датчиков — используется DataCollector и REST API |
| `44333` | HTTPS    | Веб-интерфейс                                                     |

Если эти порты уже заняты, измените их в `docker-compose.yml` и в `Config/appsettings.json`:

```yaml
ports:
  - '44331:44330'   # порт хоста : порт контейнера
  - '44334:44333'
```

```json
"Kestrel": {
  "SensorPort": 44330,
  "SitePort": 44333
}
```

**Примечание:** Конфигурация Kestrel определяет порты, которые сервер слушает внутри контейнера. Левая сторона маппинга Docker — это то, что вы открываете на хосте.

---

## Обновление

Для обновления до последней версии:

```bash
# Загрузить новый образ
docker pull hsmonitoring/hierarchical_sensor_monitoring:latest

# Перезапустить через Docker Compose (данные сохраняются в томах)
docker compose down
docker compose up -d
```

Или через `docker run` — остановите и удалите старый контейнер, затем снова выполните команду `docker run`. Тома не затрагиваются.

## Первый вход

После запуска сервера откройте `https://localhost:44333`.

* Имя пользователя: `default`
* Пароль: `default`

Перейдите в **Account → Settings** и смените пароль, прежде чем делать что-либо ещё.
