# Иерархический мониторинг датчиков (HSM)

HSM — это самостоятельная платформа мониторинга для сбора данных с датчиков ваших сервисов и приложений, оснащённая мощной и гибкой системой оповещений.

Основная ценность HSM — это движок оповещений. Вы задаёте правила (например, когда загрузка ЦП превышает 90%, когда сервис перестаёт отправлять данные или когда меняется статус), и HSM отправляет уведомления в Telegram, предоставляя полный контроль над расписаниями, шаблонами и пороговыми значениями.

---

## Что делает HSM

* Собирает метрики и статусы из ваших приложений через клиентскую библиотеку NuGet или REST API
* Организует датчики в иерархическое дерево: Продукт → Папка → Датчик
* Оценивает политики оповещений для каждого входящего значения в реальном времени
* Отправляет уведомления в чаты Telegram при срабатывании условий
* Отображает историю датчиков и текущий статус в веб-интерфейсе
* Поддерживает Grafana в качестве внешней панели мониторинга

---

## Архитектура

```text
Ваше приложение
  └── HSMDataCollector (NuGet)
        └── HTTPS ──► Сервер HSM (порт 44330)
                          ├── Движок оповещений
                          ├── Хранилище LevelDB
                          └── Веб-интерфейс (порт 44333)
                                └── Telegram Bot → Ваши чаты
```

* **Сервер HSM** — служба ASP.NET Core. Принимает данные датчиков, оценивает политики, хранит историю, обслуживает веб-интерфейс и управляет ботом Telegram.
* **HSMDataCollector** — пакет NuGet для приложений .NET. Собирает системные метрики (ЦП, память, диск) и отправляет пользовательские значения датчиков на сервер.
* **REST API** — HTTP-эндпоинты для любого языка или скрипта, который не может использовать пакет NuGet.

## Статус датчиков на первый взгляд

| Значок | Статус | Значение |
| :---: | :--- | :--- |
| 🟢 | **Ok** (ОК) | Активных оповещений нет |
| 🔴 | **Error** (Ошибка) | Сработала хотя бы одна политика оповещения |
| 🕑 | **OffTime** (Отключен) | Датчик не отправлял данные в течение окна TTL |
