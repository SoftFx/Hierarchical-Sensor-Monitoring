# Registration

This page describes how to register a new user in HSM and get started with the monitoring system.

See also: [Users](Users), [Glossary](Glossary), [Using HSM Site](Using-HSM-Site)

---

## First Connection

To start working in the monitoring system, the user needs to register.

![image](https://github.com/SoftFx/Hierarchical-Sensor-Monitoring/assets/43994777/8693d1e6-0fb1-413e-ad3c-4047fc932306)

After clicking the **"Register"** button, a form will open in which the user must enter the Login and Password. This Login and Password will be used during authorization. Password minimum length is 8 characters. After entering the login and password, you must click the **"Submit"** button to create a user.

![image](https://github.com/SoftFx/Hierarchical-Sensor-Monitoring/assets/43994777/3002386b-eca5-4756-a21b-05170f7773ea)

In case of successful registration, the user will be authorized with their username and password. The site will look like this:

![image](https://github.com/SoftFx/Hierarchical-Sensor-Monitoring/assets/43994777/4839c582-d194-4421-a4d6-e6e4a6cd3ccc)

This happens because the user has not been added to any product yet. A user can have 3 roles on the website:

| Role | Description |
|---|---|
| **Product Viewer** | Can view products and sensor data in the monitoring tree, but cannot edit settings |
| **Product Manager** | Can manage products, access keys, and invite other users |
| **Admin** | Has full access to all products, users, and server configuration |

See also: [Glossary → Roles](Glossary)

After obtaining the appropriate role for one of the products, it will be displayed on the tree. Example below:

![image](https://github.com/SoftFx/Hierarchical-Sensor-Monitoring/assets/43994777/912c340e-099f-4411-a42b-bda8a5144bd5)
