# Начало работы

HSM — это самостоятельная платформа мониторинга. Вы разворачиваете один сервер, подключаете к нему свои приложения и получаете оповещения в реальном времени при возникновении проблем.

---

## Как это работает

```text
Ваше приложение
  └── HSMDataCollector (NuGet)  ──►  Сервер HSM  ──►  Telegram
       или REST API                      │
                                        └──► Веб-интерфейс (графики, история, оповещения)
```

1. Ваше приложение отправляет значения датчиков — метрики, статусы, любые числовые или текстовые данные
2. Сервер HSM принимает их, хранит историю и оценивает правила оповещений для каждого значения
3. При срабатывании правила — отправляется уведомление Telegram в настроенные чаты
4. Вы также можете просматривать все данные и управлять настройками в веб-интерфейсе

---

## Ключевые понятия

- **Продукт** — контейнер для одного приложения или сервиса. Содержит все свои датчики и ключи доступа.
- **Датчик** — отдельный поток данных, идентифицируемый по пути (например, `MyApp/database/query_time_ms`). Создаётся автоматически при первой отправке данных в новый путь.
- **Ключ доступа** — аутентифицирует данные, поступающие из вашего приложения. Создаётся для каждого продукта.
- **Папка** — объединяет продукты в группы. Чаты Telegram подключаются на уровне папки.
- **Политика оповещений** — правило для датчика: если значение > 90 → отправить сообщение в чат Telegram.
- **TTL** — если датчик перестаёт отправлять данные → срабатывает оповещение OffTime.
