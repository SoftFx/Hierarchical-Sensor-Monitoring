# Справочник условий оповещений

Это полная справка о том, какие условия доступны для каждого типа датчиков.

---

## Обзор типов датчиков

| Тип | Значение | Bar? |
|-----|----------|------|
| Bool | true / false | Нет |
| Int | целое число | Нет |
| Double | число с плавающей запятой | Нет |
| String | текст | Нет |
| TimeSpan | длительность | Нет |
| Version | номер версии (например, 1.2.3) | Нет |
| Rate | число с плавающей запятой, отображается как скорость | Нет |
| Enum | целое число, сопоставленное с именованными значениями | Нет |
| IntegerBar | агрегированный int за временное окно | Да |
| DoubleBar | агрегированный double за временное окно | Да |
| File | содержимое файла | Нет |

## Комбинация условий

Несколько условий в одной политике объединяются логикой AND или OR:

* **AND** — все условия должны быть истинными для срабатывания оповещения
* **OR** — любое одно истинное условие запускает оповещение

---

## Типы целей

Цель — это значение, с которым сравнивается свойство:

| Тип цели | Описание |
|----------|----------|
| Константа | Фиксированное значение, которое вы вводите (например, > 90.0) |
| Последнее значение | Предыдущее значение того же свойства (например, "оповещение, если значение изменилось на любую величину") |

Цель "последнее значение" полезна для обнаружения любых изменений без указания порога.

---

## EMA (Экспоненциальное скользящее среднее)

Свойства EMA сглаживают краткосрочные пики, придавая больший вес недавним значениям. Используйте их для оповещения об устойчивых трендах, а не о мгновенных колебаниях.

* **EmaValue** — EMA скалярного значения
* **EmaMin, EmaMax, EmaMean** — EMA агрегатов бара

EMA вычисляется на стороне сервера при поступлении новых значений.

---

## Операции статуса

Все типы датчиков поддерживают условия на основе статуса:

| Операция | Срабатывает, когда |
|----------|-------------------|
| IsOk | Текущий статус — Ok |
| IsError | Текущий статус — Error |
| IsChanged | Статус изменился относительно предыдущего значения (в любом направлении) |
| IsChangedToOk | Статус изменился с Error/OffTime на Ok |
| IsChangedToError | Статус изменился с Ok/OffTime на Error |

Условия статуса полезны для уведомлений о восстановлении (например, отправить оповещение, когда ошибка устранена).
