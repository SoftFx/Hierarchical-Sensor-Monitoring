# Шаблоны оповещений

Шаблоны оповещений позволяют определить набор политик оповещений один раз и автоматически применить их к нескольким датчикам на основе шаблона пути и типа датчика.

Вместо ручной настройки оповещений на каждом датчике вы создаёте шаблон — и любой датчик, соответствующий пути и типу шаблона, наследует его политики автоматически.

---

## Когда использовать шаблоны

* У вас много датчиков с одинаковыми требованиями мониторинга (например, все датчики ЦП должны оповещать при > 90%)
* Вы хотите обеспечить единообразные правила оповещений в продукте или папке
* Вы регулярно добавляете новые датчики и хотите, чтобы они настраивались автоматически

---

## Управление шаблонами

Перейдите в Alerts → Templates в веб-интерфейсе HSM.

Шаблоны привязаны к папке — они применяются только к датчикам в продуктах, принадлежащих этой папке.

---

## Поля шаблона

| Поле | Описание |
| :--- | :--- |
| Name | Отображаемое имя шаблона |
| Folder | К датчикам какой папки применяется этот шаблон |
| Path pattern | Шаблон в стиле glob для сопоставления путей датчиков |
| Sensor type | Тип датчика для сопоставления или любой тип |
| Alert policies | Одно или несколько условий (как в обычных оповещениях датчиков) |
| TTL | Необязательный тайм-аут неактивности |

---

## Сопоставление шаблона пути

Шаблон пути использует подстановочные знаки в стиле glob для сопоставления с полными путями датчиков.

| Шаблон | Совпадает с |
| :--- | :--- |
| `MyProduct/Services/*/cpu` | Любые датчики с именем cpu на один уровень ниже Services |
| `MyProduct/**/memory` | Любые датчики memory где угодно под MyProduct |
| `*/process/cpu` | Любой продукт, затем process/cpu |
| `MyProduct/worker-*/status` | Датчики типа worker-1/status, worker-prod/status |

Сопоставление происходит с полным путём датчика, включая имя продукта.

---

## Фильтр типа датчика

Шаблон можно ограничить конкретным типом датчика (Bool, Int, Double и т.д.) или установить на любой тип для сопоставления всех датчиков по заданному пути независимо от типа.

| Настройка | Поведение |
| :--- | :--- |
| Конкретный тип (например, Double) | Применяется только к датчикам Double, соответствующим пути |
| Любой тип | Применяется ко всем типам датчиков по соответствующим путям |

---

## Как применяются шаблоны

При создании датчика или изменении его пути HSM проверяет все шаблоны в папке датчика:

1. Полный путь датчика соответствует шаблону пути шаблона?
2. Тип датчика соответствует типу датчика шаблона (или это "Любой тип")?
3. Продукт датчика находится в папке шаблона?

Если все три условия совпали, политики шаблона применяются к датчику.

> Шаблоны применяются автоматически. Вам не нужно вручную назначать их датчикам.

---

## Шаблоны vs ручные политики

Датчик может иметь как политики, применённые через шаблон, так и добавленные вручную. Они сосуществуют независимо.

Если шаблон обновляется, изменения автоматически распространяются на все соответствующие датчики.

---

## Примеры сценариев использования

**Оповещение о высокой загрузке ЦП для всех сервисов:**
* Шаблон пути: `*/services/*/cpu`
* Тип датчика: `Double`
* Условие: `Value > 90`
* Шаблон: `"[$product] $sensor высокая загрузка ЦП: $value%"`

**TTL-оповещение для всех heartbeat-датчиков:**
* Шаблон пути: `**/alive`
* Тип датчика: `Bool`
* TTL: `5 минут`
* Сообщение: `"[$product] $path не отвечает"`

**Статус ошибки на любом датчике в критичном продукте:**
* Шаблон пути: `CriticalProduct/**`
* Тип датчика: `Любой тип`
* Условие: `Status = Error`
* Сообщение: `"🔴 [$product] $sensor: $status — $comment"`
